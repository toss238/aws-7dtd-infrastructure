# 7 Days to Die (7DTD) — サーバー構築ガイド

## 1. 構成の意図

本構成は、以下の設計方針に基づいて設計されている。

### コスト最適化

ゲームサーバーにはEC2スポットインスタンスを採用し、稼働コストを最小化する。踏み台サーバー（Bastion）は t3.micro の最小スペックで常時起動とし、スポットインスタンスの中断時にも管理経路を維持する構成としている。

### セキュリティの担保

ゲームサーバーはプライベートサブネットに配置し、インターネットからの直接アクセスを遮断する。外部からの通信はすべて Bastion の iptables によるポート転送（DNAT）を経由する構成とすることで、攻撃面を最小限に抑えている。

### 環境のポータビリティ

Docker Compose を採用することで、AWS 以外の環境への移行や設定のカスタマイズを容易に行えるよう配慮している。

### インフラのコード管理（IaC）

CloudFormation テンプレートは GitHub リポジトリで管理されている。テンプレートファイルを修正してコミットすると、CloudFormation の Git Sync 機能により、対応するスタックが自動的に検知・更新される。 インフラの変更はすべてコードの変更として記録されるため、変更履歴の追跡と再現性が保証される。

## 2. スタックの設定方法（CloudFormation Git Sync）

AWS コンソールの Git Sync 機能を利用してスタックを作成する。作成時には、以下の 2 つのパスを指定する。

| 設定項目                   | 説明                                     |
| -------------------------- | ---------------------------------------- |
| リポジトリのルート         | GitHub リポジトリの接続設定              |
| テンプレートファイルのパス | 各スタックに対応する YAML ファイルのパス |

### 作成順序

スタック間に依存関係があるため、必ず以下の順序で作成すること。

1. network スタック — テンプレートパスに network.yml を指定して作成する。VPC・サブネット・ルーティングが構成される。
2. bastion スタック — テンプレートパスに bastion.yml を指定して作成する。踏み台インスタンスおよびポート転送ルールが構成される。
3. game-server スタック — テンプレートパスに game-server.yml を指定して作成する。スポットインスタンスおよびセキュリティグループが構成される。

### 自動更新の動作

各 YAML ファイルを編集してリポジトリにコミット・プッシュすると、対応するスタックが自動的に更新される。例えば game-server.yml のインスタンスタイプを変更してコミットすると、スタックの変更セットが自動生成・適用される。手動での再デプロイは不要である。

## 3. Docker 環境の構築

インフラ構築完了後、ゲームサーバー上で Docker コンテナを起動する。

### 前提条件

- Bastion 経由で SSH 接続が可能な状態であること
- ゲームサーバーインスタンスが起動済みであること

### 参考リポジトリ

本プロジェクトの docker-compose.yml は、以下のリポジトリをベースに構成されている。

| 項目   | URL                                                                                                            |
| ------ | -------------------------------------------------------------------------------------------------------------- |
| 参考元 | [vinanrra/Docker-7DaysToDie (GitHub)](https://github.com/vinanrra/Docker-7DaysToDie/blob/master/docs/usage.md) |

環境変数・ボリューム設定の詳細は上記 URL を参照のこと。

### セットアップ手順

Bastion 経由でゲームサーバーに SSH 接続し、以下の手順を実行する。

```
// 1. 作業ディレクトリの作成
$ mkdir -p ~/7dtd/data/{saves,lgsm-config,serverfiles,log,backups}
$ cd ~/7dtd

// 2. docker-compose.yml を配置する
// Git リポジトリから取得するか、または手動で作成すること

// 3. コンテナの起動
$ sudo docker compose up -d

// 4. 起動ログの確認（正常起動を確認する）
$ sudo docker compose logs -f
```

コンテナが正常に起動すると、ゲームサーバーのプロセスが開始される。初回起動時はサーバーファイルのダウンロードが行われるため、完了まで数分を要する。

### docker-compose.yml の変更と反映

docker-compose.yml を修正した場合は、以下のコマンドで変更を反映させる。

```
cd ~/7dtd
sudo docker compose pull // イメージの最新化（必要な場合）
sudo docker compose up -d // 差分のみ再作成・再起動
```

## 4. ゲームクライアントからの接続

ゲームサーバーの起動完了後、クライアントから以下の手順で接続する。

```
1. ゲームを起動し、「ゲームに参加」→「サーバーを直接追加」を選択する。
2. 接続先に {Bastion の PublicIP}:26900 を入力する。
3. 接続が確立されると、ゲームワールドへの参加が完了する。
```

> [!TIP] Bastion の DNAT 設定により、ポート 26900 への外部アクセスはプライベートサブネット上のゲームサーバーへ転送される。ゲームサーバーの IP アドレスを直接公開する必要はない。
